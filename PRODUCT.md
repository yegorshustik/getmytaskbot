# Get My Task — Product Knowledge

## Что это

Telegram-бот, который превращает голосовые и текстовые сообщения в структурированные задачи. Задачи автоматически классифицируются по матрице Эйзенхауэра, получают дату/время и синхронизируются с Google Calendar.

**Целевая аудитория:** люди, которым неудобно вручную заводить задачи — проще надиктовать или написать текстом.

---

## Технологии

- **Python** + python-telegram-bot
- **Groq AI** — извлечение задач из текста + транскрибация голоса (Whisper)
- **Google Calendar API** — OAuth 2.0 + PKCE, синхронизация событий
- **SQLite** (users.db) — хранение пользователей, задач, аналитики
- **aiohttp** — веб-сервер (порт 8080) для OAuth callback и статичных страниц
- **APScheduler** — фоновые задачи (утренний дайджест, напоминания)
- **pytest** — unit-тесты на чистую логику (38 тестов)

## Основные флоу

### Онбординг
1. `/start` → выбор языка (ru / en / uk)
2. Показывается главное меню
3. При первой задаче — предложение подключить Google Calendar

### Создание задачи
1. Пользователь пишет текст или отправляет голосовое (до 60 сек)
2. Groq извлекает задачи: название, квадрант, дата, время, причина
3. Если дата/время не указаны явно → бот спрашивает "Когда планируешь?"
4. Каждая задача — отдельная карточка с кнопками

**Правила извлечения задач (промпт):**
- Каждый title обязан содержать глагол-действие
- При перечислении объектов через запятую с одним глаголом ("протестировать X, Y и Z") → всегда одна задача
- Явное время: "сегодня", "завтра", "через N минут/часов", конкретное число → вычисляется и подставляется
- Без явного времени → `suggested_date: null`, `suggested_time: null` → бот уточняет

### Кнопки на карточке задачи

**Если Google Calendar подключён:**
```
[✅ Сохранить]  [❌ Пропустить]
```
- "Сохранить" → задача идёт и в бот, и в Google Calendar автоматически

**Если Google Calendar НЕ подключён:**
```
[📅 Google Calendar]  [🍎 Apple Calendar]
[✅ Сохранить]  [❌ Пропустить]
```
- "Google Calendar" → запускает подключение (OAuth)
- "Сохранить" → только в бот

### Уточнение даты/времени
Если пользователь не назвал дату или время:
1. Бот показывает карточки задач с явными датами (если есть)
2. Для задач без даты задаёт один вопрос: "📅 Когда планируешь это сделать?"
3. Пользователь отвечает ("завтра в 15:00", "через час" и т.д.)
4. Бот разбирает ответ и показывает карточки с заполненной датой

### Конфликт в календаре
Если на выбранное время уже есть событие:
```
[✅ Сохранить всё равно]  [❌ Не сохранять]
```
- "Сохранить всё равно" → сохраняем и в бот, и в Calendar
- "Не сохранять" → полностью удаляем карточку и диалог

### Единая точка входа — выбор календаря

В `/settings` одна кнопка с тремя состояниями:

| Состояние | Кнопка | Действие |
|-----------|--------|----------|
| Ничего не подключено | `📅 Подключить календарь` | Экран выбора: Google / Apple |
| Google подключён | `✅ Google Calendar — отключить` | Отключает Google, возвращает к выбору |
| Apple подключён | `🍎 Apple Calendar — отключить` | Открывает экран управления Apple |

Одновременно может быть подключён только один календарь. После отключения любого — возврат к экрану выбора.

### Подключение Google Calendar (OAuth)
1. Генерируется ссылка с PKCE (state + code_verifier в БД)
2. Пользователь авторизуется у Google
3. Callback `/oauth/callback` → сохраняем токен, `calendar_connected=1`
4. Если были pending-задачи — показываем их снова с кнопками подключённого режима

### Apple Calendar (iCal-подписка)
Два режима:
- **Разовый `.ics`** — при создании задачи без подключённого календаря, кнопка `🍎 Apple Calendar` на карточке задачи отправляет файл для ручного добавления события
- **Живая подписка** — через `/settings → 📅 Подключить календарь → 🍎 Apple Calendar`:
  1. Пользователь нажимает "Подключить Apple Calendar" → генерируется `ical_token`
  2. Кнопка "Открыть в Apple Calendar" → `https://.../ical-open/{token}` → редирект на `webcal://...` → iOS предлагает подписаться
  3. Apple Calendar сам тянет фид раз в час → задачи появляются автоматически
  4. "Отключить подписку" → `ical_token = NULL` → фид возвращает 404 → бот показывает инструкцию как удалить подписку из iOS (Настройки → Приложения → Календарь → Учётные записи) → показывает экран выбора календаря

**Важно:** токен создаётся только при явном нажатии "Подключить", не при просмотре настроек. При отключении старая подписка в приложении остаётся (удалить с сервера нельзя), поэтому показываем инструкцию пользователю.

## Цели (SMART Goals)

### Что такое цель в боте
Бот распознаёт масштабные намерения ("Хочу похудеть к лету", "Запустить продукт за месяц") и оформляет их как SMART-цели с дедлайном и критерием успеха.

### Классификация цель vs задача
Groq-промпт с жёсткими правилами:
- Встречи, обеды, звонки, поездки → **всегда задача**
- При сомнении → **задача**
- Дедлайн не угадывается — только если явно указан в тексте

### Создание цели
1. Пользователь пишет/надиктовывает масштабное намерение
2. Groq определяет: это цель или задача
3. Если цель — бот показывает **превью** извлечённых дедлайна и критерия + кнопки «Да / Нет»
4. После «Да» — показывается онбординг «Как создать цель» (ru/en/uk)
5. Бот задаёт уточняющие вопросы (дедлайн / критерий успеха) если нужно
6. Цель сохраняется в БД

### Онбординг перед созданием цели
Показывается один раз после нажатия «Да, создать цель», только если нужны уточняющие вопросы:
```
🎯 Как создать цель

Хорошая цель — это три составляющих:
• Что: конкретный результат — «запустить онлайн-курс»
• Когда: реальный дедлайн — «до 1 сентября»
• Критерий: как поймёшь, что достиг — «первые 10 оплат»
```

### Переименование цели
Бот не поддерживает переименование напрямую. При попытке (ключевые слова: «переименуй цель», «измени название цели» и т.д.) отвечает подсказкой удалить и создать заново. Определяется по ключевым словам (без LLM).

### Привязка задач к целям
- После сохранения задачи бот проверяет совпадение ключевых слов с активными целями
- Если совпадение найдено — предлагает привязать: `[✅ Привязать] [➡️ Нет]`
- "Привязать" → сообщение редактируется в "✅ Привязано к цели…" (без дубля)
- "Нет" → сообщение с вопросом удаляется (карточка уже была показана)

### Просмотр целей (`/goals`)
- Каждая цель — отдельное сообщение-карточка:
  ```
  📌 Название цели
  до 30 апреля 2026
  ████░░ 60% — 3/5 задач
  
  [🗑 Удалить цель]
  ```

### Удаление цели
Нажатие "🗑 Удалить цель" → карточка редактируется в вопрос:
```
Удалить цель "…"?
Что делать со связанными задачами?

[🗑 Удалить задачи]  [🔗 Оставить задачи]
[❌ Отмена]
```
- "Удалить задачи" → удаляет цель + все привязанные задачи
- "Оставить задачи" → удаляет цель, задачи открепляются (goal_id = NULL)
- "Отмена" → карточка восстанавливается с кнопкой удаления
- Если после удаления активных целей не осталось — бот автоматически присылает промпт создать новую цель (`goals_empty`)

## Напоминания

### Утренний дайджест
- Каждый день в `reminder_time` (по умолчанию 08:00) по таймзоне пользователя
- Показывает задачи на сегодня (или ближайшие, если на сегодня нет)

### Напоминания о событиях Calendar
- Проверка каждые 5 минут
- За `reminder_minutes` до начала события (5 / 15 / 30 / 60 / 120 мин)
- Дедупликация — повторные напоминания не отправляются (таблица `sent_reminders`)
- При получении напоминания пользователь видит кнопки: **✅ Сделано** / **📅 Перенести**
  - "Перенести" открывает выбор: **Через час** / **Через 3 часа** / **На завтра** (время вычисляется от текущего момента с учётом таймзоны)

### Ре-вовлечение (re-engagement)
- Пользователи, неактивные 3+ дня, получают мотивирующее сообщение в 10:00 по местному времени
- Максимум 3 сообщения на пользователя
- Дедупликация через `sent_reminders`

## Админ-панель (только для владельца)

Кнопка **⚙️ Admin** появляется в главном меню только у `BOT_OWNER_ID`.

Команда `/admin` (она же `/stats`) выводит:
- Всего пользователей / с Google Calendar / с отключёнными уведомлениями
- DAU / WAU / MAU
- Новые пользователи за 7 и 30 дней (по `registered_at` — реально новые)
- Задачи и голосовые за периоды
- Полный список пользователей в моноширинном шрифте, отсортированный по `last_active DESC`
  - Формат строки: `[GM] @username — 22 Apr 2026` (G=Google Calendar, M=уведомления выкл)

При запуске бота владелец получает уведомление: `✅ Get My Task Bot запущен`

## База данных

### `users`
| Поле | Описание |
|------|----------|
| `chat_id` | Telegram ID |
| `lang` | Язык (ru/en/uk) |
| `calendar_token` | JSON токен Google OAuth |
| `calendar_connected` | 0/1 |
| `timezone` | Напр. `Europe/Moscow` |
| `reminder_time` | HH:MM |
| `reminder_enabled` | 0/1 |
| `reminder_minutes` | Минут до события |
| `pending_task_json` | Задачи на паузе (ждут подключения календаря) |
| `last_active` | Последняя активность |
| `ical_token` | Токен для Apple Calendar iCal-фида |
| `last_reengagement` | Дата последнего re-engagement сообщения |
| `reengagement_count` | Счётчик отправленных re-engagement |

### `tasks`
| Поле | Описание |
|------|----------|
| `chat_id` | Владелец |
| `title` | Название |
| `description` | Описание |
| `quadrant` | Q1–Q4 |
| `quadrant_name` | Локализованное название |
| `suggested_date` | YYYY-MM-DD или NULL (не указана) |
| `suggested_time` | HH:MM или NULL (весь день / не указано) |
| `done` | 0/1 |
| `synced_to_calendar` | 0/1 |
| `google_event_id` | ID события в Google Calendar |
| `task_url` | URL из описания задачи (Zoom, Meet и т.д.) |
| `goal_id` | FK → goals.id, NULL если не привязана |

### `goals`
| Поле | Описание |
|------|----------|
| `id` | PK |
| `chat_id` | Владелец |
| `title` | Название цели |
| `criteria` | Критерий успеха |
| `deadline` | YYYY-MM-DD |
| `quadrant` | Q2 (по умолчанию) |
| `status` | `active` / `deleted` |
| `created_at` | Дата создания |

### `events` (аналитика)
Типы событий: `new_user`, `text_task`, `voice_task`, `calendar_connected`

### `oauth_states`
Хранит state + code_verifier для PKCE. Записи старше 1 часа удаляются.

## Тестирование (TDD)

```bash
make test      # запустить все тесты вручную
make hooks     # активировать pre-commit хук (новая машина)
```

- **38 unit-тестов** в `tests/` на чистую логику: `build_tasks_by_day`, `format_date`, `adjust_task_to_future`, `_progress_bar`, `_task_matches_goal`, `is_goal_rename_intent`, `fmt_title`, TEXTS
- **Pre-commit хук** — тесты запускаются автоматически при каждом `git commit`. Если упали — коммит блокируется
- `tests/conftest.py` мокирует тяжёлые зависимости (telegram, groq, google) — тесты не требуют credentials
