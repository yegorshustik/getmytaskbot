# Claude Code — правила работы с проектом Get My Task

## Читай документацию перед любой задачей

Перед началом разработки всегда читай:

- **`PRODUCT.md`** — продуктовая логика: флоу, правила классификации, UX, команды бота
- **`ARCHITECTURE.md`** — структура кода: модули, слои зависимостей, следующие шаги рефакторинга

Если задача затрагивает продуктовое поведение (новая функция, изменение флоу) — сначала `PRODUCT.md`.  
Если задача техническая (рефакторинг, новый модуль, тесты) — сначала `ARCHITECTURE.md`.

---

## Архитектурные правила (из ARCHITECTURE.md)

1. **Зависимости только вниз**: `db.py` ← `calendar_utils.py` ← `bot.py`. Нижние слои не импортируют из верхних.
2. **DB-запросы только в `db.py`**: нет `sqlite3.connect()` в bot.py или calendar_utils.py (исключение — inline-запросы в callback-хендлерах, помечены как tech debt).
3. **Строки только в `texts.py`**: новые пользовательские тексты добавляются в TEXTS сразу на трёх языках (ru/en/uk).
4. **TDD-guard**: любая чистая функция покрывается тестом перед коммитом; `make test` перед деплоем.
5. **Явные импорты**: никаких `from module import *`.

---

## Рабочий процесс

```bash
make test     # проверить тесты
make deploy   # задеплоить на сервер
make logs     # смотреть логи
make status   # статус сервиса
```

Pre-commit хук запускает тесты автоматически — коммит заблокируется при падении.

---

## После любого изменения

- Если изменилась продуктовая логика → обновить `PRODUCT.md`
- Если изменилась структура кода / появился новый модуль → обновить `ARCHITECTURE.md`
- Если добавлены новые ключи в TEXTS → убедиться, что все три языка покрыты (тест `test_texts.py`)
